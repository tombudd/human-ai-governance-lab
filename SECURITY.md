# Security Policy

This repository is public and educational. Please do not submit secrets, credentials, private logs, customer data, real receipts, private prompts, or sensitive system details in public issues or pull requests.

## Reporting Sensitive Issues

If you find a security issue, contact:

**tom@tombudd.com**

Please include:

- a clear description of the issue
- safe reproduction steps, if available
- the affected file or behavior
- whether any secret, token, credential, private data, or real receipt is involved

## Do Not Publish

Do not publish:

- secrets or credentials
- private prompts or schemas
- production logs
- customer data
- real workflow receipts
- proprietary architecture
- internal governance labels
- deployment details

## Scope

This repository contains toy examples and deterministic public logic only. It is not a production workflow system or a disclosure of private infrastructure.
